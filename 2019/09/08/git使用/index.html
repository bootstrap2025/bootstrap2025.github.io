前言

记录下git里常用的命令!

reflog命令

reflog 可以看到提交的版本记录。

reset命令

reset命令可以将git的HEAD指针移动到指向的地方，这样如果reset的分支没有push到远端，这样处理是没有问题的，但是如果代码已经push到远端，就必须 java git push -f origin 远端分支，这样是强制提交，这样做有个问题 ，就是多人协作的话，这种提交会把这个分支后面的版本全部覆盖掉，就会造成问题，如果想要恢复最新的分支，需要备份最新的版本号，但是这样在多人协同处理情况下有覆盖代码的风险，所以已经push到远端的分支不建议使用reset分支，这个时候就考虑使用git revert命令。
注意:git reset时出现 git Unstaged changes after reset 可以采用 git stash git stash drop 解决。

revert命令

对于git revert命令时，对于 A->B->C->D 其中C和D是错误版本的情况下，要使用revert命令时需要： revert D ，revert C 相当于一个一个的备份，这样整个git 的版本就类似于 A->B->C->D-D’->C’ 可以看到都是新增版本，并没有向前移动指针，这样是安全的，但是如果有很多版本回退，可以采用 git revert OLDER_COMMIT^..NEWER_COMMIT’，并且git revert命令是可以直接使用git push 到远端分支的，这样风险就小很多。开发过程中应该尽量使用git revert命令，尽量不要使用git reset命令。
